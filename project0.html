HTML - Hypertext Markup Language (the heart of the web)


HTML is made up of text content, which is what you see. It is made up of markup, which is what that content looks like or how it is arranged. It is made up of references to other documents, like images and videos, or any other files. And it can be made up of links to other pages.
Plain text is plain text in HTML, for the most part. But if we want to make the text look different, we need to use something called markup, which is one of the other facets of HTML.
Tags
HTML markup is made up of things called tags. The way a tag is structured: you have got, a less than symbol, the tag name, a greater than symbol, the tag contents, which can actually be other tags or it could just be text, another less than symbol, a slash, the name, and a greater than symbol.
The first one is the opening tag, the last one, with the slash, is the closing tag. The whole thing is an element.
Bold tag everything that is between these tags appears bold.
emphasis tag everything that is between these tags appears italic.
Html can be nested.

Computes are stupid

HTML attributes


Attribute is a property of an HTML element.
We still have our opening tag name and, and a closing tag but we have an attribute that have a name and that is equal a value. The value appears inside cotes that are inside the opening tag, after de equal sign.
Tags can have multiple attributes.
An example of a tag that uses an attribute is the anchor tag (a tag).
Anchors are for making links. We have the a of the opening anchor tag followed by the name of the attribute (href), the equal sign and the value inside cotes. The value in this case is an URL. After the opening tag with the attribute and the value, we have the contents and the ending tag with the slash and the a. The ending tag is the same as the ones of the elements without attributes. The content will be the only thing that will appear at the browser, and it will be a link to the URL put as the value.

The image tag (img).


The image tag (img) is for including images. It has the following structure. The opening IMG tag. This is an example of how the web service can serve all sorts of different types of content It has an attribute called source, SRC equals the URL. This is the URL of the image to download. And then there is another attribute called alt. Which stands for alternate, and this is text that gets displayed when the image does not load. If our browser requested a URL that does not exist, this is the text that gets displayed, and it is also for blind people.
Every tag that we have talked about so far has had a closing tag. Images do not. This is the entire tag. There is no content to an image. This is called a void tag.
A void tag has no content, and because it has no content, it does not need a closing tag.
Images just appear in line with text.

Whitespace


When we enter text in the, in the editor, if we put text on multiple lines, when we render it in HTML it render all on a single line.
In HTML all whitespace, new lines, tabs, spaces all turn into a single space. To force our HTML to have multiple lines we can use another tag called the br tag.
Br stands for break, and it is a void tag. And we put this after any lines that we want to appear on multiple lines. And we can even enter multiple br tags if we want multiple line breaks.
Another way of doing line breaks is to use the p tag. This stands for paragraph. This is not a void tag. So the p tag has content and because it is not a void tag, it has a opening tag, the content and the closing tag, and this will render as one paragraph.

Inline versus Block


The br tag is what we call inline, and the p tag, is what we call block.
The br tag just end a line.
The br tag just basically says the line ends here and the following will appear at the next line.
The p tag works differently. The p tag actually makes an invisible box.
So instead of just rendering text, blank line, what this is actually rendering is an invisible box.
And that is what the p tag does. And this invisible box can have height, and it can have width, where inline elements are just text.
Inline and block elements have kind of a different behavior.
So far, all the elements we have learned other than p are inline elements.
b, em, br, even img are inline elements.
So far, the only block element we have learned is the p element.

Container elements


Elements that do nothing but contain stuff inside of them. These elements can be used to create web pages with complex structures.

span and div


Span and div are both normal elements.
They can both have content.
And the only difference between these two is, span is inline, and div is block.
These elements do not do anything other than just contain their content. And there is a way to attach styles to them, to adjust different behaviors of how they display.
CSS it is a separate language for adding styles to your documents or changing how different text looks.
We will be using spans and divs a lot for controlling how text is laid out.
But the important thing to remember is that spans are inline and divs are block.

The structure of an HTML document


Now, what we have been seeing before is just little markup.
But an actual complete HTML document has quite a bit more in it.
First we have the DOCTYPE; what kind of HTML. This used to be a lot more complex. But now that we are using HTML5, it has a nice clean simple doctype, which is just the string HTML. Very simple.
Then we have a tag. This is the basically the opening HTML tag and the closing HTML tag. This surrounds the entire rest of the document. That is all it does, is it surrounds the entire document.
Now we have other tag, the head. This part of the document has metadata and another random stuff. If we were to include JavaScript and CSS, that would be included in the head section.
Now we also have the tittle tag, and this is the title of the page. This is what would appear in the top of your browser window, or in, in your browser tab, the content of the tittle tag.
Then we have the body tag, and the body tag is the actual contents of the document. And basically, we have been, we have been working just in the body tag in this lesson. And for most of this course, we are really just going to be focusing on generating the content that fits between the body tags.
All this other stuff is important but it does not change very often and, it is pretty simple, but it is pretty important for it to be there. All of the interesting stuff though happens between the body tags.
An HTML document has a head section and that is where we have our title, references to any CSS files, and Java script files, and that sort of stuff. And then we have the body section and this is where the actual contents of the document, the stuff that we see on the screen. This course where mostly going to focus on the body section and there is a lot of content that can go in there.
